# 레이어 추가, 삭제, 순서 조절

## 결정 요약

- 레이어 추가는 타일이 없는 빈 일반 레이어를 만든다.
- 일반 레이어와 CV payload가 있는 레이어는 동일한 레이어로 취급한다.
- 새 레이어의 기본 이름과 id는 `Layer N` / `layer-n` 형식을 사용한다.
- 새 레이어는 최하단에 추가한다.
- 레이어 목록의 위쪽은 캔버스 최상단을 의미한다.
- 레이어 삭제는 확인 다이얼로그를 거치며, 마지막 1개 레이어는 삭제할 수 없다.
- 레이어 순서 조절은 각 레이어 행의 위/아래 버튼으로 수행한다.
- 추가, 삭제, 순서 변경은 기존 레벨 스냅샷 히스토리에 기록해 Undo/Redo 대상에 포함한다.

## 구현 범위

- `src/stores/layers.ts`에 레이어 생성, 정렬, 순서 정규화, 이동, 삭제 헬퍼를 둔다.
- 레이어 목록은 `order` 내림차순으로 표시하고, 저장 시 `order`를 `0..n-1`로 정규화한다.
- 최상단 레이어의 위 이동 버튼, 최하단 레이어의 아래 이동 버튼, 마지막 레이어의 삭제 버튼은 비활성화한다.
- 레이어를 삭제하면 레이어 안의 타일도 함께 제거되지만, 팔레트 타일은 전역 자원으로 남긴다.
- 삭제한 레이어가 active 또는 selected 상태였다면 목록 기준 가까운 위 레이어를 우선 선택하고, 없으면 아래 레이어를 선택한다.

## 검증

- `pnpm run build`
- `pnpm run biome`
- 수동 확인 항목:
  - `Add Layer`가 최하단 레이어를 만들고 바로 active/selected 상태로 만드는지 확인한다.
  - 위/아래 버튼이 목록 순서와 캔버스 렌더링 순서를 함께 바꾸는지 확인한다.
  - 마지막 레이어 삭제가 막히는지 확인한다.
  - 삭제 확인 다이얼로그의 취소/확정 동작을 확인한다.
  - active/selected 레이어 삭제 후 가까운 위 레이어로 선택이 이동하는지 확인한다.
  - 추가/삭제/순서 변경이 Undo/Redo로 복원되는지 확인한다.

## 질문과 답변 요약

- 레이어 추가는 어떤 레이어를 만들까? 빈 일반 레이어를 만들되 CV 레이어와 일반 레이어는 payload 유무 차이로 보고 하나의 레이어 개념으로 취급하기로 했다.
- 새 레이어는 어디에 둘까? 최하단에 추가하기로 했다.
- 삭제 안전장치는 어떻게 둘까? 확인 다이얼로그를 띄우고 마지막 1개 레이어는 삭제하지 못하게 하기로 했다.
- 순서 조절 UI는 어떻게 둘까? 각 레이어 행에 위/아래 버튼을 두기로 했다.
- 목록 위쪽은 어떤 의미일까? 목록 위쪽이 캔버스 최상단을 의미하도록 하기로 했다.
- 새 레이어 기본 이름과 id는 어떻게 만들까? `Layer N` / `layer-n` 형식을 사용하기로 했다.
- 삭제 후 선택은 어떻게 할까? 가까운 위 레이어를 우선 선택하고, 없으면 아래 레이어를 선택하기로 했다.
- 레이어 관리 작업도 히스토리에 포함할까? 추가, 삭제, 순서 변경 모두 Undo/Redo 대상에 포함하기로 했다.
- CV payload가 있는 레이어 삭제 시 팔레트도 정리할까? 레이어만 삭제하고 팔레트는 전역 자원으로 남기기로 했다.
