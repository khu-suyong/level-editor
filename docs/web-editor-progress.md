# 웹 에디터 구현 진척도 보고서

- 작성 기준일: 2026-05-13
- 대상 프로젝트: Sketch2Map - CV기술을 활용한 스케치 기반 언리얼엔진 레벨 생성
- 평가 범위: 현재 `level-editor` 레포의 웹 에디터 구현체 전체와 미커밋 작업 트리 변경사항
- 기준 문서: `26-1 프로젝트계획서_심수용.docx`

## 1. 계획서 기준 웹 에디터 목표

프로젝트 계획서에서 Sketch2Map은 다음 네 단계의 파이프라인으로 정의되어 있다.

1. 사용자가 규칙 기반 스케치를 입력한다.
2. CV 기술이 스케치 정보를 해석하여 구조화된 JSON 데이터로 변환한다.
3. 웹 에디터에서 변환된 JSON 데이터를 시각적으로 확인하고 수정한다.
4. Unreal Engine 연동 도구가 JSON 데이터를 Blueprint Actor 배치 규칙으로 변환하여 `.umap` 레벨을 생성한다.

현재 레포의 웹 에디터는 이 중 3단계에 해당한다. 따라서 핵심 목표는 CV 인식 결과로 생성된 레벨 JSON을 불러와 사용자가 시각적으로 검토하고, 오인식된 객체를 추가·삭제·수정한 뒤 이후 Unreal Engine 연동 단계로 넘길 수 있는 데이터를 만드는 것이다.

## 2. 현재 구현 완료 항목

현재 구현체는 웹 에디터 프로토타입의 기본 조작 기반을 상당 부분 갖추고 있다.

| 구분 | 구현 상태 | 근거 |
| --- | --- | --- |
| 앱 기반 구조 | 완료 | SolidJS, Vite, TypeScript 기반 앱 셸과 라우터가 구성되어 있다. |
| UI 시스템 | 완료 | `@suis-ui/kit`, `@vanilla-extract/css`, Lucide 아이콘을 사용한 패널·툴바 UI가 구현되어 있다. |
| 캔버스 렌더링 | 완료 | PixiJS `Application`, `Container`, `Graphics` 기반으로 그리드와 타일을 렌더링한다. |
| 반응형 캔버스 크기 | 완료 | `ResizeObserver`로 호스트 크기 변화에 맞춰 renderer를 resize한다. |
| 레벨 데이터 기본 모델 | 부분 완료 | `LevelData`, `LevelLayer`, `TilePlacement`, `TileMapping`과 Zod 검증 스키마가 있다. |
| 상태 관리 | 완료 | `nanostores` 기반으로 에디터 상태, 히스토리, 레벨 상태 저장소가 분리되어 있다. |
| 기본 편집 도구 | 완료 | 선택, 브러시, 지우개, 팬 도구가 구현되어 있다. |
| 타일 추가·삭제 | 완료 | 브러시 드래그로 타일을 추가하고 지우개 또는 삭제 명령으로 제거할 수 있다. |
| 선택 편집 | 완료 | 클릭 선택, 드래그 박스 선택, Shift 추가 선택, 선택 영역 이동이 구현되어 있다. |
| 복사·붙여넣기 | 완료 | 선택 타일을 내부 클립보드와 브라우저 클립보드에 복사하고 원하는 셀에 붙여넣을 수 있다. |
| 실행 취소·다시 실행 | 완료 | add, delete, replace, move 액션을 기록하고 undo/redo로 되돌릴 수 있다. |
| 뷰 조작 | 완료 | 마우스 휠/트랙패드 스크롤, Ctrl/Meta 줌, Space/팬 도구/중간 버튼 팬, 키보드 팬을 지원한다. |
| 컨텍스트 메뉴 | 완료 | 우클릭 메뉴에서 복사, 붙여넣기, 삭제, 선택 해제, 뷰 초기화를 실행할 수 있다. |
| 속성 패널 | 부분 완료 | 줌 슬라이더와 그리드 크기 표시가 있다. |
| 레이어 패널 | 부분 완료 | 레벨명, 레이어 목록, 레이어별 타일 좌표 표시 UI가 있다. |

구현의 중심은 `src/components/pixi-viewport/*`, `src/stores/history.ts`, `src/stores/editor.ts`, `src/models/level.ts`, `src/pages/_components/*`에 있다.

## 3. 부분 구현 및 미완성 항목

계획서가 요구한 “CV 결과 JSON을 확인·수정하는 웹 에디터” 관점에서는 아직 다음 항목이 남아 있다.

| 항목 | 현재 상태 | 영향 |
| --- | --- | --- |
| JSON 불러오기 | 미구현 | CV 모듈이 생성한 JSON 파일이나 문자열을 웹 에디터에 주입할 수 없다. |
| JSON 내보내기 | 미구현 | 수정된 레벨 데이터를 Unreal Engine 연동 단계로 전달할 수 없다. |
| CV 결과 수신 흐름 | 미구현 | 스케치 인식 결과와 웹 에디터가 아직 연결되어 있지 않다. |
| 도메인 객체 모델 | 미구현 | 계획서의 플랫폼, 위험 요소, 적, 일반 오브젝트, 플레이어 시작 위치가 별도 타입으로 표현되지 않고 단일 타일 배치로만 표현된다. |
| 타일·객체 타입 선택 | 부분 구현 | 브러시는 기본 `tileId`만 사용하며 사용자가 타일 종류를 선택하는 UI가 없다. |
| 레이어 선택 동작 | 부분 구현 | 상위 컴포넌트에는 `onSelectLayer` 흐름이 있으나 현재 레이어 항목 클릭은 확장 토글 위주라 활성 레이어 변경과 연결되지 않는다. |
| 저장·불러오기 | 미구현 | 브라우저 세션 밖으로 편집 결과를 보존하는 기능이 없다. |
| 검증·오류 표시 | 부분 구현 | Zod 스키마는 있으나 사용자에게 JSON 검증 오류를 보여주는 UI는 없다. |
| 테스트 | 미구현 | 단위 테스트, 컴포넌트 테스트, E2E 테스트 구성이 확인되지 않는다. |
| Unreal Engine 연동 | 범위 밖/미구현 | 현재 레포에는 JSON to `.umap` 연동 도구가 포함되어 있지 않다. |

## 4. 현재 품질 및 검증 상태

현 시점의 기능 구현은 프로토타입 수준으로 확인되지만, 제출 가능한 빌드 상태는 아니다.

| 검증 명령 | 결과 | 주요 원인 |
| --- | --- | --- |
| `pnpm run build` | 실패 | `Slider` 컴포넌트 props 타입 불일치, `PropertyPanel`의 미사용 import·변수로 TypeScript 빌드가 실패한다. |
| `pnpm run biome:check` | 실패 | `PropertyPanel` 미사용 import·변수 경고와 `Slider`, `PropertyPanel` formatter 위반이 있다. |

문서 작성 작업에서는 코드 수정을 하지 않았으므로, 위 검증 결과는 현재 작업 트리의 기존 상태를 반영한다.

## 5. 진척도 평가

| 평가 범위 | 추정 진척도 | 판단 근거 |
| --- | --- | --- |
| 웹 에디터 요구 범위 | 약 50% | 캔버스 표시, 선택, 추가, 삭제, 이동, 복사·붙여넣기, undo/redo 등 핵심 편집 조작은 구현되어 있다. 그러나 JSON 입출력, CV 결과 연결, 도메인 객체 모델, 저장 기능이 없어 계획서의 웹 에디터 목표를 완전히 충족하지는 못한다. |
| 전체 Sketch2Map 파이프라인 | 약 20-25% | 웹 에디터 조작 기반은 진행되었지만, 스케치 입력 규칙, CV 인식 모듈, JSON 변환 모듈, Unreal Engine 자동 배치 도구, 최종 `.umap` 생성 흐름은 현재 레포에서 확인되지 않는다. |

위 수치는 기능 항목 기반의 정성적 추정이다. 실제 진척률을 정량화하려면 전체 요구사항 목록, 각 기능별 가중치, 외부 CV/Unreal Engine 모듈의 별도 구현 상태를 함께 반영해야 한다.

## 6. 종합 판단

현재 웹 에디터는 “타일 기반 레벨을 시각적으로 조작하는 브라우저 편집기 프로토타입”으로는 의미 있는 수준까지 구현되어 있다. 특히 PixiJS 기반 캔버스 렌더링, 마우스·키보드 입력 처리, 선택 영역 이동, 편집 히스토리, 컨텍스트 메뉴 등 실제 에디터 사용감에 필요한 기반 기능이 이미 마련되어 있다.

다만 프로젝트 계획서의 최종 목표는 단순 타일 편집기가 아니라 “CV로 생성된 구조화 JSON을 검토·보정하고 Unreal Engine 레벨 생성으로 전달하는 중간 도구”이다. 이 기준에서는 JSON import/export, CV 결과 연결, 게임 도메인 객체 타입 체계, 저장·검증 UI가 아직 빠져 있으므로 현재 상태는 중간 단계의 웹 에디터 프로토타입으로 보는 것이 적절하다.

우선순위는 빌드·Biome 실패를 먼저 정리한 뒤, JSON 입출력과 객체 타입 선택 UI를 연결하는 것이다. 그 다음 CV 결과 JSON 샘플을 실제로 불러와 편집하고 다시 내보내는 최소 경로를 완성하면 계획서의 웹 에디터 단계에 대한 기능적 검증이 가능해진다.
